# Variable Handling

When writing your PHP code you will find that very often you need to know if a variable exists, if it has been previously created or if the value inside a variable is empty.

Let's say we are going to build a log in page for our website. The user is supposed to fill in a form that has two different input fields - one called username and one called passwd. We want to make sure that the form is being submitted to us with both of those form fields, that the form was submitted using the POST method and that neither of those two fields were left blank.

# isset

The isset() method checks to see if a variable exists. It could be a stand alone variable or it could be checking for a single item within an array.

# empty

The empty() method checks to see if a string variable is null or an empty string.

if ( isset($_POST['username']) && isset($_POST['passwd']) && !empty($_POST['username']) && !empty($_POST['passwd']) ) { 
  //write the code here that handles the log in.
} else {
  //write the code here to tell the user that they did not submit a proper form
}

The double ampersands mean AND. We are testing to make sure that both $_POST parameters exist AND that neither is empty.

While often the order in which we put the checks with isset() or empty() does not matter. However, in this case there was a purpose to the order. When you string together a series of tests with && it means that you need all of them to return true. As soon as one of them returns false the testing stops. They are read from left to right. If the first one fails then none of the other tests are required. In our code above we want to make sure that both $_POST['username'] and $_POST['passwd'] exist. If one of them does not exist within the $_POST array then there is no point in checking to see if it is empty.
